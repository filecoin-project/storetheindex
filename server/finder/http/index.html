About

Content routing is the a term used to describe the problem of finding providers for a given piece of content. If you have a hash, or CID of some data, how do you find who has it?

In IPFS, A DHT is used as a decentralized answer to content routing. When considering the number of records present in current filecoin storage providers, it has become clear that simply announcing these records in the DHT will place an undue burdon on current DHT participants. It will take too much bandwidth, and storage space to be practical

This project encompasses a number of work streams aiming to increase the scalability and resilience of content routing. Network Index nodes can provide aggregated views of content routing, and harness the heterogeneity of network resources. Delegated Content Routing defines protocols for multiple content routing providers to be discovered and used safely, efficiently, and in a decentralized manner by clients.

Status

Last Updated: Jan 27th

Store The Index: A network index node storing a mapping of CID to provider. Status: You're Using it!.

Index Provider: A library for providing advertising provided content over the indexer 'ingestion protocol'. Status: Test instance running and validated. Integration into lotus markets validated on local storage provider. Next step: validation with larger set of storage providers.

Retrieval Client: A small client for retrieving content using a network indexer for content routing. Status: Manual success for individual graphsync retrievals against index provider instance.

Delegated Routing: A library for delegating content routing to an HTTP provider. Compatible with the store the index content routing API. Status: prototype for individual node testing.

Content Router Discovery Protocol: A design for how delegated routers are automatically discovered and distributed in an IPFS environment. Status: Proposal
